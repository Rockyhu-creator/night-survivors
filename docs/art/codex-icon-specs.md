## 0. 管线与通用约定

- **绘制方式**：由 engineering-lead 在 `gen_assets.py` 中用 `new_canvas(S)` + `px / rect / fill_ellipse_shaded` 程序化绘制，最后 `save(img, "<key>.png", 2)` 输出。
- **画布与输出**：分类卡 / 祭坛按钮图标 **40×40 画布，scale=2 → 80×80 PNG**；`codex_book` **48×48 画布，scale=2 → 96×96 PNG**（体量略大于分类卡）。
- **透明背景**：使用 RGBA 透明画布，所有非透明像素经 `out_line()` 加 1px 深色描边 `OUTLINE = (8,4,14,255)`。UI 中再以 CSS `image-rendering: pixelated` 缩放显示；你感知到的 32~48px 是 CSS 显示尺寸，与现有 `altar_*` 完全一致。
- **统一风格**：暗色背景透明、清晰描边、哥特夜裔；仅用几何形状 + 像素块，无照片级细节。
- **资产归属**：这 5 个 key **不在** `AI_OWNED` 集合中，程序化生成器会正常写出（不会被判 skip），无需特殊处理。

## 1. `codex_artifacts` —— 神器图鉴卡 icon

- **用途与位置**：图鉴一级菜单 `codex-hub` 第一张分类卡（`id=artifacts`，配色标签 `gold`）。当前 `ui.js` 的 `showCodex()` 引用了不存在的 `art_sword`，需改为 `codex_artifacts`。
- **主体造型（像素可绘制）**：一枚**暗金符文圆盘 + 中央悬浮的棱面钻石**。圆盘为环形（外环 gold、内环略暗），环上 8 个等距小刻痕（rune ticks）；中心一颗竖向菱形钻石（faceted：左上受光面、右下暗面、中央高光芯）；钻石与环之间留一圈透明，制造「悬浮圣物」感。
- **配色**：圆盘 gold D/M/L；钻石主面 M、暗面 D、高光 L；刻痕 L。
- **建议尺寸**：40×40 @2x = 80×80，透明背景（与 `altar_*` 同级）。
- **关键视觉元素清单**：
  1. 环形符文圆盘（gold）
  2. 中央菱形钻石（faceted，3 面分色）
  3. 环上 8 向 rune 刻痕
  4. 钻石核心高光
  5. 钻石与环之间的悬浮留白
- **与现有资产区分点**：**不复用任何 `art_*` 神器贴图**（尤其 `art_storm` 三刃风轮、`art_matrix` 八芒星、`art_sepulcher` 紫光棺木）；也不直接套用 `gem_*` 经验宝石造型（本 icon 是带金盘的「圣物徽记」，非裸钻石）。采用「圆盘 + 悬浮钻」而非「武器 / 闪电 / 棺木」造型。
- **绘制提示**：`fill_ellipse_shaded` 画外环（环形掩膜：仅保留 `r∈[12,16]`）；中心用 `rect`/`px` 画菱形（对角线分面法同 `gen_gem`）；环上 `for k in range(8)` 循环 `px` 打刻痕。

## 2. `codex_monsters` —— 怪物图鉴卡 icon

- **用途与位置**：`codex-hub` 第二张分类卡（`id=monsters`，配色标签 `purple`）。当前引用 `icon_skull`，需改为 `codex_monsters`。
- **主体造型（像素可绘制）**：**恶魔眼印记（horned sigil）**。一个纵向尖椭圆 / 菱形紫框（上尖下尖的 lozenge）；顶部两侧各一只外曲的**犄角**（用斜像素阶梯绘制，非骷髅头骨）；框中央一道竖立的**发光裂瞳**（细长竖缝 + 上缘高光，眼色青紫）；底部两枚小獠牙暗示。整体是「印记 / 图腾」感，而非完整头颅。
- **配色**：紫框 purple D/M/L；眼瞳 soul 青 `(200,245,255)` 主 + 高光 `(230,245,255)`；犄角用 bone 暗调或 purple 亮调。
- **建议尺寸**：40×40 @2x = 80×80，透明背景。
- **关键视觉元素清单**：
  1. 紫框 lozenge 印记
  2. 顶部双犄角（差异化关键）
  3. 中央发光裂瞳
  4. 底部小獠牙
  5. 角根 / 框缘高光
- **与现有资产区分点**：**不复用 `icon_skull` 作为唯一元素**（`icon_skull` 是完整正面颅骨 + 红眼窝 + 牙列，本 icon 只取「角 + 眼」做图腾化变形，无颅骨轮廓、无牙列横排）；也不复用敌人精灵（`enemy_bat` 膜翼、`enemy_skeleton` 持剑骷髅）。强调「角 + 裂瞳」而非「头骨」。
- **绘制提示**：lozenge 用 `for y in range` 算 `half` 画对称框；犄角用两条 `px` 阶梯线从框顶向斜上外扩；裂瞳用窄 `rect` + 顶缘 `px` 高光。

## 3. `codex_weapons` —— 武器图鉴卡 icon

- **用途与位置**：`codex-hub` 第三张分类卡（`id=weapons`，配色标签 `red`）。当前引用 `weapon_blade`，需改为 `codex_weapons`。
- **主体造型（像素可绘制）**：**交叉兵刃（剑 + 斧 成 X）**。两条对角线交叉：左上为长剑（剑身窄长、剑尖朝左上、护手 + 短柄），右下为战斧（弯刃朝右下、木柄）；交叉点一枚红色铆钉 / 魂芯发光。整体红调钢质，刃口带 red 边缘光。
- **配色**：钢 steel D/M/L（同 `weapon_blade`）；红边缘光 red M/L；木柄 `(96,60,32)`；中心魂芯 red L 发光。
- **建议尺寸**：40×40 @2x = 80×80，透明背景。
- **关键视觉元素清单**：
  1. 左上长剑（斜置）
  2. 右下战斧（斜置）
  3. 中心红铆钉 / 魂芯
  4. 刃口 red 边缘光
  5. 护手 / 木柄点缀
- **与现有资产区分点**：**不复用 `weapon_blade`（单把苦无）为唯一元素**；也区别于 `weapon_axe`（单斧）、`weapon_aura`（红环 + 六芒星）、`altar_dual` 与 `passive_rage`（两把**相同**匕首交叉）。本 icon 用「剑 + 斧**异型**交叉」+ 红色辉光，辨识点在于双武器类型不同。
- **绘制提示**：两条对角线各用像素斜线（`for i in range` 沿 `dx=dy` 步进）画剑身 / 斧刃；斧刃用 `px` 弧阶；中心 `fill_ellipse_shaded` 小球做发光铆钉。

## 4. `codex_book` —— 图鉴一级菜单书本 icon（入口标识）

- **用途与位置**：`codex-hub` 屏的图鉴总入口标识（建议置于标题「游戏图鉴」左侧 / 上方，作为 masthead 徽记）。体量略大于三张分类卡 icon。
- **主体造型（像素可绘制）**：**闭合的暗色魔典（正面偏 3/4）**。中性炭灰封面（**非紫**），中央一道**金属搭扣 / 锁扣**（gold 或银），书脊在左、封面右侧露出**发光书页**（淡蓝白辉光沿页缘）。可加一道顶部高光与底部投影暗示体积。
- **配色**：封面 neutral stone D/M/L（**非紫，关键区分**）；搭扣 metal gold D/M/L（或银 `(180,180,200)`）；书页辉光 `(200,230,255)` / `(230,245,255)`。
- **建议尺寸**：**48×48 @2x = 96×96**，透明背景（略大于分类卡）。
- **关键视觉元素清单**：
  1. 中性暗色书封（非紫，关键区分）
  2. 中央金属搭扣 / 锁扣
  3. 露出的发光书页
  4. 书脊与封面分面
  5. 顶部高光 / 底部投影
- **与现有资产区分点**：**不复用 `passive_tome`**（那是紫色封面 + 金边 + 符文竖纹的魔法书，且为单色紫）。本 icon 用中性炭灰封面、加金属搭扣、强调发光书页，色彩与造型均不同；也区别于 `chest`（宝箱）与 `art_devour`（圣杯）。
- **绘制提示**：`rect` 画封面主体 + 左侧书脊窄条（更亮一档）；封面中央 `rect` 画搭扣（横向金条 + 中点扣环）；右侧 / 底部用半透明 `px`(alpha≈150) 画发光页缘。

## 5. `altar_menu` —— 灵魂祭坛主菜单按钮 icon

- **用途与位置**：主菜单 `btn-altar`（「灵魂祭坛」）左侧 `menu-btn-icon`，当前复用 `altar_hp`。需改为 `altar_menu`。
- **主体造型（像素可绘制）**：**石台祭坛 + 上方悬浮灵魂火**。一座**阶梯式方坛 / 石台**（2~3 层收窄的方块台基，暗石质，左侧来光、右侧暗面）；坛顶中央一道刻纹 / 凹槽；坛**正上方悬浮一团幽蓝 / 紫灵魂火**（上尖下宽的火焰轮廓 + 中心亮芯 + 几点火星），与石台间留透明间隙。
- **配色**：石 stone D/M/L（同 `decal` 石质但更暗）；灵魂火 soul 蓝紫 D/M/L（或蓝 `(90,140,220)`/`(170,200,255)`）；刻纹 / 高光用 soul L。
- **建议尺寸**：40×40 @2x = 80×80，透明背景（与 `menu-btn-icon` 现有按钮同级）。
- **关键视觉元素清单**：
  1. 阶梯式石台（方坛 / 台基）
  2. 坛顶刻纹 / 凹槽
  3. 上方悬浮灵魂火（蓝紫）
  4. 灵魂火亮芯 + 火星
  5. 石台体积光（左亮右暗）
- **与现有资产区分点**：**不复用 7 个 `altar_*` 升级图标**（`altar_hp` 心形护符、`altar_spd` 风翼、`altar_dmg` 滴血獠牙、`altar_gain` 鬼脸、`altar_dual` 交叉双匕、`altar_slot_weapon` 符文牌+剑、`altar_slot_passive` 符文牌+盾）——那些都是**单属性符号**，本 icon 是**完整「祭坛建筑」结构**（石台 + 悬浮火），造型维度完全不同；也不复用 `chest` / `decal_cross`（十字墓碑）。
- **绘制提示**：台基用 3 层 `rect`（自底向上逐层收窄、左亮右暗分档，同 `gen_altar_*` 附近 `rect` 分色法）；灵魂火用 `for y` 算半宽画上尖下宽轮廓 + 中心 `fill_ellipse_shaded` 亮芯 + 几点 `px` 火星。
