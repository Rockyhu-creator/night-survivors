## 0. 管线与通用约定（对齐现有工程）

- **绘制方式**：由工程负责人在 `gen_assets.py` 中用 `new_canvas(S)` + `px / rect / fill_ellipse_shaded` 程序化绘制，最后 `save(img, "<file>.png", 1)` 输出（`scale=1`，与现有敌人/Boss 一致；分类图标用的 `scale=2` 不适用于本批）。
- **透明背景**：RGBA 透明画布，所有非透明像素经 `outline()` 加 1px 深色描边 `OUTLINE = (8,4,14,255)`（同 `assets.js` 与 `codex-icon-specs.md`）。
- **统一风格**：暗色背景透明、清晰描边、哥特夜裔；仅几何形状 + 像素块，无照片级细节；与现有 `bat / skeleton / slime / elite / boss_*` 协调但明显区分。
- **资产归属**：本批次新增的 `enemy_shadow_hunter.png / enemy_gargoyle.png / boss_avatar.png` **不在** `AI_OWNED` 集合中，程序化生成器会正常写出，无需特殊处理。
- **改动映射**：详见文末「落地清单 / 资产改动映射」。

## 一、重绘优先级清单

### 优先级总表

| 优先级 | 条目 | 现状问题 | 是否强制 | 改动类型 |
|---|---|---|---|---|
| **P0** | `avatar`（永夜化身） | `sprite: 'boss_overlord'`，与 mid-Boss overlord **完全撞脸**（最严重） | ✅ 必须 | 独立新贴图 `boss_avatar`，脱离 overlord |
| **P0** | `shadow_hunter`（暗影猎手） | `sprite: 'bat'` + `tint:'#9b59b6'`，仅染紫；**且 `tint` 字段在渲染路径未应用**，实为纯蝙蝠 | ✅ 必须 | 独立新贴图 `enemy_shadow_hunter`，脱离 bat |
| **P0** | `gargoyle`（石像鬼） | `sprite: 'elite'` + `tint:'#7f8c8d'`，仅染灰；`tint` 同样未应用，实为纯精英 | ✅ 必须 | 独立新贴图 `enemy_gargoyle`，脱离 elite |
| **P1** | `volatile`（爆破词缀） | 叠加在 bat/skeleton/slime 等，`e.affix` 无任何外观标识，玩家无法预判爆炸 | ✅ 必须（低成本） | **不换主体**，渲染叠加橙色脉冲裂纹/引线辉光 |
| **P1** | `shielded`（护盾词缀） | 同上，减伤 70% 无外观标识 | ✅ 必须（低成本） | **不换主体**，渲染叠加蓝色结晶护盾光环 |
| **P2** | `elite`（精英） | 已有独立贴图，用户觉「造型普通/辨识度低」——待评测 | ⚠️ 评估，可选强化 | 评估 + 可选加「精英光环」信号 |
| **P2** | `baron / queen / overlord` | 三者共用 `_boss_base` 人形长袍骨架，仅配色+头冠区分，小尺寸下可能「只是颜色不同」 | ⚠️ 评估，不强制 | 评估 + 可选差异化建议 |

### 判定结论（供主理人拍板）

- **P0 三项必须重绘**：撞脸 / 复用问题确凿，且与现有美术基调不符（染紫蝙蝠、染灰精英在「夜裔」世界观里不成立）。
- **P1 两项必须做、但低成本的叠加层**，不动主体贴图，纯渲染提示（见第三节）。
- **P2 两项属「评估 / 可选」**：
  - `elite` 已有独立造型（带翼铠甲恶魔+金冠），与 bat/skeleton/slime 区分度足够；仅建议「加一道脉动的精英光环」作为高威胁信号，**不强制重绘**。
  - `baron/queen/overlord` 当前配色+头冠已能区分，**判定为可接受的差异化，不强制重绘**；仅当 Phase 2 决定强化 Boss 演出时，再按建议做轮廓差异化。

### 2.1 `avatar`（永夜化身）— P0

- **用途与位置**：终局 Boss（15 分钟降临，击杀=通关）。当前 `BOSSES[3].sprite: 'boss_overlord'`，与 mid-Boss `overlord` 100% 撞脸。
- **造型（像素可绘制）**：**悬浮的虚空神祇，非人形态**。下半身无双腿，化入翻涌的**暗夜星云裙裾**（像素块 + 点状星芒，向外渐隐）；上半身类人但「破碎」——躯干由暗色晶片/裂隙拼成，正中一道**纵向发光裂瞳**（青白→紫）；双肩展开的是**星尘翼**（离散点状星芒，而非 overlord 的实体肩翼）；头顶**破碎的虚空光环**（环裂成数段，缺口处溢光），取代 overlord 的宽王冕。整体轮廓明显大于且「飘」于 overlord（更宽、无支撑腿），一眼可辨为「神祇 / 虚空」而非「人类君王」。
- **配色**：暗夜紫黑 D/M/L（表末行）；星辉/裂瞳 灵魂青 D/M/L；星尘点 `(150,90,255)`→`(210,170,255)`；裂隙能量纹 红紫 `(200,60,120,255)`（作体内能量裂痕）。
- **尺寸**：绘制画布 **S=72, scale=1 → 72×72 PNG**（对齐 `_boss_base` 的 S=64 约定并略放大以容纳星云细节）；`data.js` 中 `avatar.spriteSize` 保持 **168**。
- **关键视觉元素（6 项）**：
  1. 悬浮无腿的暗夜星云裙裾（向外渐隐）
  2. 头顶破碎虚空光环（缺口溢光）
  3. 躯干纵向发光裂瞳（青白→紫）
  4. 双肩星尘翼（点状星芒，非实体翼）
  5. 躯干裂隙能量纹（红紫）
  6. 外溢星辉粒子（环绕浮动）
- **不复用（明确）**：**不复用 `boss_overlord`**（彻底重画，非染色 / 非换冠）；也不复用 `_boss_base` 人形长袍骨架。
- **绘制提示**：裙裾用 `for y` 算半宽 + 随机 `px` 星点（alpha 渐隐）；光环用 `for a in range` 画断续圆弧、缺口处留透明；裂瞳用窄 `rect` + `fill_ellipse_shaded` 高光芯；星尘翼用散布 `px` 星点（带 `lighter` 辉光感）；整体经 `outline()` 描边。

### 2.2 `shadow_hunter`（暗影猎手）— P0

- **用途与位置**：永夜阶段解锁（unlockAt:540）的后期怪。当前 `ENEMY_TYPES.shadow_hunter.sprite: 'bat'` + `tint:'#9b59b6'`，实为纯蝙蝠（tint 未在渲染路径生效）。
- **造型（像素可绘制）**：**半虚影的猎手剪影**。类人形但半透明虚化（像素块间刻意留透明缝隙，制造「影」感）；**前倾冲刺姿态**（呼应其 dash 行为）；双手生成一把**紫色能量弓**（或能量利爪）；身后拖曳**飘逸暗影尾迹**（数条向后渐隐的紫色拖影）；面部为冷光裂瞳（与 avatar 区分：横向细缝单点冷光，而非纵向大裂瞳）。
- **配色**：暗影紫 D/M/L（表末 purple 行）；能量弓/利爪 亮紫 `(180,110,255,255)` / 高光 `(225,180,255,255)`；尾迹 `(110,55,170,255)` 向后渐隐；裂瞳 `(200,160,255,255)`。
- **尺寸**：绘制画布 **S=40, scale=1 → 40×40 PNG**（对齐 bat 的 S=34 约定并略放大以容纳人形细节）；`data.js` `shadow_hunter.spriteSize` 保持 **40**。
- **关键视觉元素（6 项）**：
  1. 半虚影猎手躯干（像素透缝）
  2. 前倾冲刺姿态（呼应 dash）
  3. 紫色能量弓 / 利爪
  4. 飘逸暗影尾迹（渐隐）
  5. 冷光裂瞳（横向细缝）
  6. 肩 / 腕暗影尖刺
- **不复用（明确）**：**不复用 `bat`**（完全独立的人形猎手造型，非蝙蝠染色）；也不复用 `skeleton`（持械骷髅）。
- **绘制提示**：躯干用 `fill_ellipse_shaded` + 间隔 `TRANSPARENT` 像素做虚化；能量弓用弧阶 `px`（同 `gen_art_spiral` 附近弧线法）+ `lighter` 辉光；尾迹用 3~4 条递减 alpha 的 `px` 拖影；姿态用躯干 `y` 偏移制造前倾。

### 2.3 `gargoyle`（石像鬼）— P0

- **用途与位置**：永夜阶段解锁（unlockAt:600）的重甲怪（immuneKnockback）。当前 `ENEMY_TYPES.gargoyle.sprite: 'elite'` + `tint:'#7f8c8d'`，实为纯精英。
- **造型（像素可绘制）**：**蹲伏的石质守卫**。折起的**岩石翼**（背后收拢、表面斑驳石纹）；**蹲伏石躯**（厚实块状、关节带裂石）；**发光眼**（暖橙裂隙光，与夜裔冷调形成对比焦点）；爪状前肢撑地；头顶短角/断角。整体石灰冷色，缝隙透出暗夜苔紫微光。
- **配色**：石灰 stone D/M/L（表末 stone 行）；裂纹/发光眼 暖橙 D/M/L（爆破橙行，作裂隙光；或冷青备选 `(120,200,220,255)`）；石纹阴影 `(40,42,50,255)`；暗夜苔紫 `(90,60,120,255)` 作缝隙微光。
- **尺寸**：绘制画布 **S=96, scale=1 → 96×96 PNG**（与 elite 同量级 S=96）；`data.js` `gargoyle.spriteSize` 保持 **96**。
- **关键视觉元素（6 项）**：
  1. 蹲伏石躯（块状、厚实）
  2. 折起岩石翼（斑驳石纹）
  3. 发光眼（暖橙裂隙光）
  4. 撑地爪肢
  5. 断角 / 棘刺
  6. 石面裂纹（透暗夜紫微光）
- **不复用（明确）**：**不复用 `elite`**（独立石质造型，非精英染色）；也不复用 `boss_*` 人形。
- **绘制提示**：石躯用 `rect` + `fill_ellipse_shaded` 块状拼接（左亮右暗分档，同 `gen_elite` 分色法）；岩石翼用 `fill_ellipse_shaded` + 扇贝下缘 `TRANSPARENT`；发光眼用 `fill_ellipse_shaded` 小芯 + `lighter` 暖橙辉光；裂纹用折线 `px`（alpha 渐隐）。

### 2.4 `elite`（精英）— P2 评估 / 可选强化

- **现状评测**：已有独立贴图（`gen_elite`，S=96）——带翼铠甲恶魔 + 金冠犄角 + 红眼，与 bat/skeleton/slime 区分度**足够**，辨识度本身不差。用户「造型普通」的直觉可能来自其**威胁信号弱**（看不出比普通怪强多少）。
- **建议（可选，不强制）**：在主体外叠加一道**脉动精英光环**（暗金/紫，亮度随威胁等级变化），作为「高威胁」的通用信号；冠冕可加更高更尖的棘刺。**不重画主体**。仅在 P0 完成后有余力时执行。
- **尺寸**：维持 S=96 / spriteSize 96，仅叠加光环层（渲染期，非贴图重画）。

---

### 2.5 `baron / queen / overlord` — P2 评估 / 不强制重绘

- **现状评测**：三者共用 `gen_assets.py` 的 `_boss_base`（人形长袍骨架：梯形下摆 + 肩 + 头 + 发光眼 + 胸前纹章），**仅**通过配色 + 头冠 + 翼/裙/领配件区分：
  - `baron`：红金长袍 + 双角王冠 + 高立领
  - `queen`：苍白蓝白长裙 + 尖顶冠 + 加宽裙摆
  - `overlord`：暗紫虚空长袍 + 巨大肩翼 + 宽王冕
  - 配色与头冠已能区分；**但核心轮廓同为人形长袍**，小尺寸下可能「只是颜色不同」。
- **判定**：**当前可接受的差异化，不强制重绘**。若 Phase 2 决定强化 Boss 演出，给差异化建议（仅建议，不列入本批次强制清单）：
  - `baron`（血色男爵）：更厚披风 + 高立领，整体「贵族吸血鬼」庄重宽肩。
  - `queen`（苍白女王）：改为**悬浮巫妖**——下身无裙摆、化幽光/骨链，更瘦高飘逸，骨冠更尖。
  - `overlord`（永夜君王）：保持暗紫 + 巨肩翼 + 宽王冕，作为 baseline 不改动。
- **尺寸**：维持各自 S=64 / spriteSize 128~156。

## 三、词缀标识实现建议（P1：volatile / shielded）

> 仅描述**视觉与叠加方式**，不给代码。目标：不换 sprite，仅在渲染期依据 `e.affix` 叠加标识层，让玩家一眼识别。
> 渲染锚点：`src/entities.js` 敌人绘制（`drawImage` 主体后，约 594 行附近），已有 `e.affix` 字段（来自 `createEnemy` 的 `affix` 参数）；项目已有 `globalCompositeOperation='lighter'` 加法辉光写法（见 `systems.js:135`、`weapons.js:721`），可复用。

### 3.1 `volatile`（爆破）— 橙色脉冲裂纹 / 引线辉光

- **叠加方式**：在怪主体**之上**叠加放射状裂纹 + 引线辉光层；不改主体 sprite。
- **视觉元素**：
  - 3~5 条从怪中心向外的**放射状裂纹**（像素折线 `px`，带分叉），模拟「即将引爆」。
  - 裂纹交汇处 / 引线端点的**橙色辉光**（小 `fill_ellipse_shaded` 亮芯，用 `lighter` 加法辉光）。
- **视觉参数**：
  - 颜色：主线 `#e67e22`（= `AFFIXES.volatile.color`），高光 `#ffb866`。
  - 脉冲频率：**~1.5 Hz**（`sin(t·2π·1.5)`），alpha 在 **0.4 ~ 0.9** 间脉动。
  - **预警增强**：当怪血量低于阈值（或死亡前 0.5s），脉冲加速到 ~3 Hz 并提亮，提示「即将爆炸」。
  - 尺寸：随 `e.radius` 缩放（裂纹长度 ≈ `e.radius * 1.2`）。
- **合成**：`globalCompositeOperation='lighter'`，叠加后恢复 `source-over`。

### 3.2 `shielded`（护盾）— 蓝色结晶护盾光环

- **叠加方式**：在怪**外围**叠加一圈结晶护盾光环；不改主体 sprite。
- **视觉元素**：
  - 环绕怪体的**六边形 / 椭圆结晶光环**（半透明蓝描边）。
  - 环上 6~8 枚**结晶尖**（朝外的三角 `px`），暗示「硬护盾」。
  - 受击（`e.flash>0`）时整环闪亮一下。
- **视觉参数**：
  - 颜色：主线 `#3498db`（= `AFFIXES.shielded.color`），高光 `#aee0ff`。
  - 环透明度：**基础 alpha 0.25 ~ 0.5**，呼吸脉冲 ~0.8 Hz。
  - `e.flash>0` 时 alpha 拉到 ~0.9 并加亮（参考 `entities.js:602` 的 flash 写法）。
  - 尺寸：环半径 ≈ `e.radius * 1.35`。
- **合成**：`globalCompositeOperation='lighter'`（辉光感），叠加后恢复 `source-over`。

> **可访问性备注**：爆破/护盾的「颜色编码」需配合形状差异（裂纹 vs 结晶环），避免仅靠色盲不可辨（见 `design/accessibility-requirements.md`，建议至少 Standard 级：形状 + 颜色双通道）。
